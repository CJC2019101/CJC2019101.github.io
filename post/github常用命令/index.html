Github常用命令

2020年1月20日
| 阅读

使用github的好处

  1. 提高工作效率，github作为一个远程版本控制仓库可以简化工作。
    如：领导意见不统一，有时程序员需要做两个功能不同的程序出来。通常情况下我只能复制两个文件分开开发，如果每个程序的功能过多很容易出错。而我们使用github 的“git branch name” 可以非常简便的解决。
  2. 存储代码。
  3. 版本控制。如果程序需要改写，可以直接回溯代码到没有添加需求的时间段。
  4. 分工合作集成管理。

github认证功能

描述： 你想过没有，为什么每次github上传代码不需要认证呢？如果不认证的话岂不是每个人只要知道了你的github上远程仓库的地址。那么每个人都可以修改你的代码。

  • SSH认证
    认证过程（个人理解）：SSH认证需要两个钥匙，一个是公钥存放放在github上，另一个是私钥存放在本地电脑上，且一个电脑只能有一个私钥。 本地仓库上传代码私钥加密，github公钥解密。
  • Https认证
    https就是每次提交代码的时候都需要输入账户名称和密码。 不推荐使用！博主自己没有测试过！
  • 注意：这里纯粹是自己为了理解记忆，描述不清极有可能错误。不影响操作！

github的本地仓库创建。

本地仓库相当于本地代码副本

  1. git init，初始化git仓库。
  2. touch README.md，创建程序具体操作、功能的描述文件。
    • 这是一个markdown文件，专门用于项目介绍。
  3. git add filename，将文件添加至本地仓库。
    • ‘.’：表示所有文件
  4. git commit -m “commmit description”，将本地仓库中改动了的代码提交至暂存空间，并添加描述。

github本地仓库与远程仓库连接

远程仓库相当于远处代码的副本，这样即使本地仓库坏了，如果我有远程仓库的地址一样可以恢复代码。

  1. git remote add origin git@github.com:<"你的github账户名称">/<"连接的远程仓库名称">.git 。 将本地仓库与github上的某个远程仓库连接对应。
    • 本地仓库没有与github远程仓库连接才调用。 而且只需要调用一次即可永久连接上github上的远程仓库。
  2. git push -u origin master 将本地的master分支推送到origin主机，同时指定origin为默认主机，后面就可以不加任何参数使用git push了。
    注意：
    • 只有创建本地仓库才可以与远程仓库连接。
    • 由于是第一次连接远程仓库，所以我们还需要认证方式。

github的使用步骤

1.clone远程仓库中的代码

描述： 实际开发中很多时候程序员都是半途接手别人的代码，而公司代码一般都使用 github、华为云、码云等远程终端存放代码。那么我们是需要将远程终端的代码 clone（下载） 下来的。

git clone <远程仓库url地址> ：下载代码，推荐使用SSH方式。

2.本地代码修改后提交到远程仓库

前提：本地仓库与远程仓库已经连接对应上。

1. git add filename :将修改了代码的文件添加至本地仓库保存，没有改动过的代码不能添加。
2. git commit -m "commit description"：将本地仓库代码上传至暂存空间。 github上传到远程仓库的代码并不是直接上传本地仓库的代码。而是将本地仓库代码上传至一个“暂存空间”，再从暂存空间提交被修改了的代码到远程仓库。 因此你没有commit 是不能成功提交代码到远程仓库的。
3. git push：将“暂存空间”中的代码提交到远程仓库。
描述：至此才算完成了github一个简单的提交代码流程。

一个完整的、基本的github流程就走完了。如果遇到其他问题根据实际情况搜索解决就好。

github高级操作

  • 命令简写
    运行以下命令即可
touch ~/ . bashrc：创建命令行配置文件
echo 'alias ga = "git add"'>> ~/ . bashrc ：流的重定向，将“ alias ga = "git add" ”追加到命令行配置文件中。 将“ git add ” alias（重命名）为 ga。这样ga就等价于git add 命令，但是不影响git add命令的使用。 以下同理。
echo 'alias gc="git commit"'>> ~/ . bashrc 
echo 'alias gl="git pull"'>> ~/ . bashrc
echo 'alias gp=" git push"'>> ~/ . bashrc
echo 'alias gco=" git checkout"'>> ~/ . bashrc ：切换分支
echo 'alias gst="git status -sb"'>> ~/ . bashrc ：展示暂存空间与本地仓库工作状态详细
source ~/.bashrc ：运行命令行配置文件使配置生效
  • 其他实用命令
git branch X : 创建分支X，直接输入git branch显示所有分支。
git checkout X ：切换分支X。
git merge X：整合X分支。
git status -sb : 简介展示本地仓库与暂存空间工作状态信息。
git clone <远程仓库url地址> ：下载代码，推荐使用SSH认证方式下载代码。
